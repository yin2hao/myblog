# 重装windows配置编程环境

因为长时间没管理空间，各种环境依赖都往C盘装。现在已经积重难返~~（强迫症犯了）~~，再加上当前使用的 win11 24H2 一堆 bug \(比如开机时键盘重复速度被随机隐性调到最低等\)，经过思考决定全盘格掉重装



因为我有点轻微的强迫症，所以就想要将所有的编程相关的放在一起，但是不同编程工具的默认行为不一样。

maven的默认仓库放在C盘下面。管理仓库位置需要手动编辑 maven 的配置文件

```XML
<!-- localRepository
 | The path to the local repository maven will use to store artifacts.
 |
 | Default: ${user.home}/.m2/repository
-->
<localRepository>D:/application/code/tools/maven/.m2</localRepository>
```

虽然在这里设置了，但一个不注意，Maven wrapper自动下载一个新的把配置覆盖了，就又放到C盘去了



gradle 和 maven 类似，也是有本地的仓库。但它与 maven 不同的是，Gradle 可以通过[环境变量规定仓库位置](https://docs.gradle.org.cn/current/userguide/directory_layout.html#dir:gradle_user_home)

|**`GRADLE_HOME`**|指定 Gradle 存储其全局配置属性、初始化脚本、缓存、日志文件等的 `GRADLE_USER_HOME` 目录。<br>如果未设置，则默认为 `USER_HOME/.gradle`。|
|---|---|
|**`GRADLE_USER_HOME`**|Gradle 的安装目录。<br>可用于指定本地 Gradle 版本，而不是使用 wrapper。<br>您可以将 `GRADLE_HOME/bin` 添加到您的 `PATH` 中，以用于特定应用程序和用例（例如测试 Gradle 的早期版本）。|



androiod studio的 android SDK 和虚拟机也会默认放在C盘，但与 Gradle 类似，可以[通过环境变量约束](https://developer.android.google.cn/tools/variables?hl=zh-cn)

|`ANDROID_HOME`|设置 SDK 安装目录的路径。设置后，该值通常不会更改，并且可以由同一台计算机上的多个用户共享。`ANDROID_SDK_ROOT` 也指向 SDK 安装目录，但已废弃。如果您继续使用它，Android Studio 和 Android Gradle 插件将检查旧变量和新变量是否一致。|
|---|---|
|`ANDROID_USER_HOME`|为 Android SDK 中包含的工具设置用户偏好设置目录的路径。 默认为 `$HOME/.android/`。<br>某些较旧的工具（例如 Android Studio 4\.3 及更低版本）不会读取 `ANDROID_USER_HOME`。如需替换这些旧工具的用户偏好设置位置，请将 `ANDROID_SDK_HOME` 设置为要在其下创建 `.android` 目录的父目录。|

Python 的 pip 和 nodejs 的 npm 都会把包放在安装位置，不会直接塞到C盘，十分好评。
